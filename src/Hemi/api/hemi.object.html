Hemi.object

  • Hemi JavaScript Framework
  • API Index

Description

The object utility class includes an API decorator for object mapping.

static Hemi.object

version 3.5.1

Convenience methods for instrumenting object array mapping and retrieval.

Index

Example Index

  • Instrumentation Example: Add an object accessor for conveniently storing and retrieving other objects.

Property Index

  • public accessorindex (internal): A hash of variant identifiers whose value is the index of accessors. The array is created for an object with implementsObjectAccessor, and is stored on the objects sub structure.
  • public accessornames (internal): A hash of variant names whose value is the index of accessors. The array is created for an object with implementsObjectAccessor, and is stored on the objects sub structure.
  • public accessors (internal): An array of variant values. The array is created for an object with implementsObjectAccessor, and is stored on the objects sub structure.
  • public objects (internal): The objects property is used throughout the Hemi JavaScript Framework to store object pointers. This property as created if it does not exist.

Method Index

  • public addNewAccessor (internal): Adds the object to the accessor array.
  • public addObjectAccessor (internal): Creates object accessors, getAccessorByName, getAccessor, getAccessors, and isAccessor methods, and the Accessors, Accessornames, and Accessorindex arrays on the objects sub structure. It is up to the implementing class to specify the property values jointly in order for the method values to resolve.
  • public addObjectDeconstructor (internal): Creates the deconstructors sigterm and destroy.
  • public clearAccessors (internal): Clears all arrays and array maps.
  • public destroy (internal): Prepares the object to be destroyed. As a decoration, this method invokes any locally defined handle_destroy handler and increments the internal ready_state to five (5). The method is created for an object with addObjectDeconstructor.
  • public getAccessor (internal): Returns the accessor for the given index. The method is created for an object with implementsObjectAccessor.
  • public getAccessorByName (internal): Returns the accessor based on the specified name. The method is created for an object with implementsObjectAccessor.
  • public getAccessors (internal): Returns the accessors array. The method is created for an object with implementsObjectAccessor.
  • public isAccessor (internal): Returns true if the accessor exists. The method is created for an object with implementsObjectAccessor.
  • public removeAccessor (internal): Removes the specified object from the accessor arrays.
  • public sigterm (internal): Sends a termination signal to the object. This automatically invokes the destroy method. The method is created for an object with addObjectDeconstructor.

addObjectAccessor

  • Top

Creates object accessors, getAccessorByName, getAccessor, getAccessors, and isAccessor methods, and the Accessors, Accessornames, and Accessorindex arrays on the objects sub structure. It is up to the implementing class to specify the property values jointly in order for the method values to resolve.

Syntax

b = addObjectAccessor( o, s )

Parameters

  • o as object : Object for which the accessors will be created.
  • s as String : Name of the accessor.

Returns

b as boolean: Returns true if the accessors were created.
