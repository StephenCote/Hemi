				<div class = "content_data">

				<h3>About the Hemi JavaScript Framework</h3>
				<div class = "content_data">
					<!--
					<p>Where is version 2?  Version 2 is <a href = "/projects/engine/">Engine for Web Applications</a>.</p>
					-->
					<p>New features in v 3.0 include a better class loader, compile-time dependency detection, the new <a href = "api/hemi.app.module.html">Module</a> library including unit testing extensions, logging utility and improved messaging space, a more generic namespace - and more.  All of the configuration fat was cut.</p>

							<ul>

Login required to save data on server.   The <i>DWAC Designer</i> includes the server-side compliments to the <i>Framework Designer</i>.  You can access <i>DWAC</i> by registering with this site, or using the <a href = "/documents/html/technical/web/coreWeb.html">Core Web</a> project.</li>
							</ul>

Performance, Behavior (aka: IMNMotion), and Experience (aka: Raccoon) monitors are available, but require a correctly configured server for configuration, shredding, aggregation, and reporting.  These modules are include in the <i>Account Manager</i> / <i>Core Web</i> projects, but the documentation for setting them up is not ready yet.</li>
					</ul>

				<h3>Setup</h3>
				<div class = "content_data">
					<ul>
						<li>Download the latest version: <a href = "/distribution/Hemi_3.0.2.zip">Hemi_3.0.2.zip</a></li>
						<li>Unzip to a directory in your Web site. By default, the Hemi root path is: /Hemi/</li>
						<li>Add Hemi to your Web page:
							<ul>
								<li>Everything all at once: &lt;script type = "text/javascript" src = "/Hemi/hemi.comp.js"&gt;&lt;/script&gt;</li>
								<li>Or, load on demand: &lt;script type = "text/javascript" src = "/Hemi/hemi.js"&gt;&lt;/script&gt;</li>
							</ul>
						</li>
						<li>If a different location is desired, set the framework path: &lt;script type = "text/javascript"&gt;Hemi.hemi_base = "/Where/Is/Hemi/";&lt;/script&gt;</li>
					</ul>
				</div>
