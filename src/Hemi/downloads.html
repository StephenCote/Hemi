Individual files

  • hemi.comp.js - everything compiled together.
  • hemi.js - The Hemi core includes xml request library (caching, pooling, JSON, and more), the Registry Service for object directory management, and Message Service for message publication and subscription.
  • Components/
    • Content Viewer - A component for displaying content as pagined data (via Rich Select).
    • Draggable - A component for making a bit of XHTML draggable.
    • Drag Tracker - A component for monitoring Draggable components.
    • Manager - A component for managing open windows.
    • Select Bus - A component for changing the bus of the hemi.data.io service.
    • Session - A component for representing the Core Web session to Hemi.
    • Steps - A component for visualizing a set of paged data (E.G.: Wizard steps or instructions).
    • Tab Strip - A component for creating a tab strip.
    • Testable - A component for instrumenting unit tests on a node.
    • Tree - A component for creating a tree control.
    • Tree Context - A component for monitoring and displaying the selected context of a tree control.
    • Wide Select - A component for displaying a buffered and paginated list control (optimized for working with large result sets).
    • Window - A component for creating a window control.
  • Fragments/
    • Active Source - Console interaction with the live DOM. Same features as the Active Source capability built into JavaScript Profiler.
    • Builder Controls - Basic toolbar controls for the Framework Designer builders.
    • Engine Stats - Tools to view and gauge the running condition of Hemi.
    • Framework API Browser - Try it - Browse the API (Note: This example loads the Fragment via a Template).
    • Framework Explorer - View of tree, tree context, and content viewer fragments.
    • Function Monitor Injection - Injectible version of JavaScript Function Monitor. Adds the function monitor code to a page. Used by Script Profiler Fragment.
    • IOGrid - Variation of Rich Select using the hemi.data.io service to marshal data requests.
    • JavaScript Profiler Injection - Injectible version of JavaScript Function Monitor. Adds the script profiler code to a page. Used by Script Profiler Fragment.
    • Member Model Tools - Utilities for modeling JavaScript constructs. Used by the Framework Designer builders.
    • Rich Select - A reusable control for drawing paginated sets of data.
    • Runtime Container - A reusable control for loading and hosting in-flight framework projects and objects during development and testing.
    • Scope Viewer - Fragment that renders the Page Scope (accessed from the Active Source UI).
    • JavaScript Profiler - Script profiling tools.
    • Tab Tools - Reusable tab controls.
    • Test Suite - A UI for running unit tests.

  • Modules/
    • console.output.js - A Module for displaying Framework messages.

  • Templates/
    • Active Source - Console interaction with the live DOM. Same features as the Active Source capability built into JavaScript Profiler.
